# 바리스타 자격증 종합 가이드 및 기출문제 플랫폼 기획안 (Blueprint)

## 1. 개요 및 목표
- **주제:** 바리스타 자격증(1급/2급) 합격을 위한 종합 가이드 및 기출문제 해설 웹사이트
- **목표:** SEO 최적화를 통한 오가닉 트래픽(월 방문자 10만) 확보, 모바일 최적화된 기출문제 풀이 UI 제공, 이탈률 최소화 및 자연스러운 애드센스 수익 창출

## 2. 사이트맵 및 계층형 메뉴 구조 (Silo Architecture)
사용자의 학습 흐름(일정 확인 -> 요약 학습 -> 실전 문제)에 맞춰 직관적인 구조로 설계.

* **홈 (Home)**: 전체 메뉴 안내, 최신 시험 일정 배너, 빠른 기출문제 시작 버튼
* **자격증 정보 (Info)**
  * 시험 일정 및 접수 방법
  * 1급 vs 2급 차이점 및 합격 기준
* **핵심 요약 (Study)**
  * 커피학 개론 요약
  * 커피 로스팅 핵심
  * 커피 추출 및 에스프레소
* **기출문제 (Exams)**
  * 2급 필기 연도별 기출문제
  * 1급 필기 연도별 기출문제
  * 모의고사 (랜덤 출제)
* **실기 팁 (Practical)**
  * 실기 평가 기준
  * 자주 하는 실수 (감점 요인)

## 3. 핵심 페이지별 화면 구성 기획 (UI/UX Layout)

### 3.1. 필기 가이드 페이지 (핵심 요약)
- **레이아웃:** 모바일 친화적인 아코디언(Accordion) 방식과 탭(Tab) 메뉴 활용.
- **가독성:** 중요한 키워드(예: 로스팅 온도, 아라비카/로부스타 특징)는 볼드 및 하이라이트 처리.
- **네비게이션:** 하단에 '다음 과목 학습하기' 버튼을 배치하여 이탈 방지.

### 3.2. 기출문제 페이지 (핵심 체류 시간 확보 페이지)
- **모의고사형 UI (1문제씩 보기 모드):** 작은 모바일 화면에서도 문제에 집중할 수 있도록 한 화면에 한 문제씩 노출.
- **정답 가리기/보기 토글 버튼:** 기본적으로 정답은 숨김 처리, '정답 및 해설 보기' 버튼 클릭 시 부드럽게(Accordion 애니메이션) 노출. Web Components를 이용하여 캡슐화된 디자인 적용.
- **진행도 바 (Progress Bar):** 상단에 `[ 5 / 50 ]` 와 같은 진행률 표시로 성취감 제공.
- **플로팅 네비게이션:** 다음/이전 문제 이동 버튼을 하단에 고정.

## 4. 애드센스 수익화 및 검색 최적화(SEO) 전략

### 4.1. 최적의 광고 배치 구역 (사용자 경험 보호)
1. **문제 풀이 중간 배너 (인피드 광고):** 기출문제 5문제 또는 10문제 풀이 후 '다음 문제로 넘어가기' 직전에 네이티브 형태의 광고 배치. 흐름을 방해하지 않고 자연스러운 클릭 유도.
2. **해설 하단 배너:** 정답 토글을 열었을 때, 해설의 가장 아래쪽에 텍스트/디스플레이 광고 배치 (사용자가 해설을 읽은 직후 시선이 머무는 곳).
3. **학습 요약 페이지 앵커 광고(모바일 하단 고정):** 화면 하단에 좁은 배너를 고정하여, 스크롤 내내 안정적인 노출 확보 (닫기 버튼 제공 필수).

### 4.2. 검색 유입을 위한 롱테일 키워드 (Long-tail Keywords) 5선
단순 '바리스타 기출문제'는 경쟁이 치열하므로, 구체적인 상황을 검색하는 타겟팅 적용.
1. "바리스타 2급 필기 기출문제 해설 다운로드 없이 풀기"
2. "바리스타 자격증 독학 합격률 높이는 커피학 개론 요약"
3. "직장인 바리스타 1급 필기 단기 속성 암기법"
4. "2024년 최신 바리스타 자격증 시험 일정 및 원서 접수 꿀팁"
5. "스마트폰으로 푸는 바리스타 모의고사 (정답 바로 확인)"

## 5. 현재 구현 계획 (코드 작성 완료 내역)
- `index.html`: 전체 레이아웃, 시맨틱 태그, Web Components(`exam-question`) 호출 적용. 사용자 요청에 따른 과소추출 문제 추가.
- `style.css`: 사용자 친화적인 CSS (그라데이션, 부드러운 그림자, 모바일 우선 반응형) 구성.
- `main.js`: 기출문제 카드를 위한 Custom Element 정의. 사용자의 디자인 피드백을 반영하여 `<details>` 스타일의 직관적인 정답 확인 UI로 개선.
